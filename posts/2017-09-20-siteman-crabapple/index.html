Global modules

Crabapple could threat headers and footers just like separate pages. So instead of one hit to SiteMgmt to get the content of the whole site, they would make 3 calls: one for header, one for “body” and one for footer.

Scheduling

We are not going to deliver pro-active cache invalidation for Crabapple in this release. They are goint to hit our delivery service several times per minute to get the data. In response, we always return the page with editorialzation data valid for given time.

Routing

In future (not for Paramount Launch) we are going to include routing in SiteMgmt.

Big topic. In future.

Editorialiaztion is optional

The site should be able to render without any editorialization. It should be “data/content driver”.

No one needs to go to SiteMgmt to make site works.

Page with Context Ref - data

We need to be able to return different editorial data for each context. So, page with code series-homepage and with context of teen-wolf should return different data than page with code series-homepage and context of catfish and different editorial data then series-homepage without context.

One level fallback is good enough? (If we cannot find page with code x and with context y, we return page with code x).
